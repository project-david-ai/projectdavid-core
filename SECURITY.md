# Security Policy

## Supported Versions

We currently only provide security updates for the latest stable release of `projectdavid-core`. Please ensure you are running the latest version before reporting a vulnerability.

| Version | Supported          |
| ------- | ------------------ |
| >= 1.28.0 | :white_check_mark: |
| < 1.28.0  | :x:                |

## Reporting a Vulnerability

**Please do not open a public GitHub Issue for security vulnerabilities.**

If you discover a security-related bug in Project David, please report it privately. Your report will be handled with high priority.

### Disclosure Process

1. **Email:** Send your report to [engineering@projectdavid.co.uk](mailto:engineering@projectdavid.co.uk).
2. **Details:** Include a summary of the issue, a proof-of-concept (if possible), and the version of `projectdavid-core` you are running.
3. **Acknowledgment:** You will receive an acknowledgment of your report within 48 hours.
4. **Resolution:** We will coordinate a fix and a release. We ask that you do not disclose the issue publicly until a patched version has been released.

## Responsible Disclosure

We are a small team (currently a solo maintainer). We appreciate your patience and your help in keeping the `projectdavid` ecosystem safe for our 280k+ users.
